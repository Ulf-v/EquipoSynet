### Puntos tratados  
- **Implementación parcial del login con Google Auth:**
  - Registro de aplicación completado en Google Cloud Console
  - Client ID y Client Secret configurados
  - Problemas con redirecciones y callback URL → parcialmente resuelto
  - Token JWT almacenado en localStorage, pero validación pendiente
  
- **Backend refactorizado:**
  - Endpoints CRUD para casos clínicos funcionando
  - Controlador de archivos (ArchivoController) implementado
  - Modelos de datos (CasoClinico) completamente actualizados con campos reales (edad, sexo, antecedentes, etc.)
  - ArchivoRepository implementa CrudRepository<Archivo, String> correctamente
  
- **Validación de endpoints en Postman:**
  - GET /api/v1/casos → retorna lista de casos (funcionando)
  - POST /api/v1/casos → crea nuevo caso (funcionando)
  - PUT /api/v1/casos/{id} → actualiza caso (funcionando)
  - DELETE /api/v1/casos/{id} → elimina caso (funcionando)
  - GET /api/v1/archivos → lista archivos (funcionando)
  
- **Base de datos MongoDB:**
  - Conexión estable confirmada con MongoDB Compass
  - Datos de prueba cargados en colección "casos"
  - Índices creados para optimizar búsquedas
  
- **Frontend rehecho:**
  - Nueva estructura con páginas separadas (chatbot.html, select_page.html, validacion_Medicos.html)
  - Scripts modularizados (burger.js, chatbot.js, validation.js)
  - CSS organizado (chatbot.css, style.css)
  - Validaciones de formularios en cliente
  
- **Problemas encontrados:**
  - Callback URL en Google Auth no se auto-redireccionaba correctamente → necesita ajuste en SecurityConfig
  - Algunos endpoints retornaban 404 por rutas mal configuradas → corregido
  - Variable de entorno GOOGLE_CLIENT_ID no se leía correctamente → solucionado con @Value
  - Frontend no lograba comunicarse con backend por Puerto diferente → resuelto con CORS actualizado
  
- **Discusiones técnicas:**
  - Debate sobre guardar token en localStorage vs sessionStorage (se opta por localStorage)
  - Discusión sobre refresh tokens y expiración de sesión
  - Consideración de validar token en cada request vs solo en acceso restringido
  - Evaluación de usar JWT vs sesiones tradicionales (se mantiene JWT)

### Decisiones tomadas  
- Completar login antes de fin de período vacacional
- Crear documentación de endpoints en Notion (con método, ruta, parámetros, respuesta)
- Implementar validación de token JWT en endpoints sensibles
- Usar .env para todas las variables sensibles (GOOGLE_CLIENT_ID, MONGODB_URI, etc.)
- Crear tests unitarios para servicios principales (antes de fin de mes)

### Riesgos identificados  
- Problemas con OAuth pueden retrasar proyecto significativamente
- Seguridad: Token JWT expuesto en localStorage (vulnerable a XSS) → requiere validación adicional
- Dependencia de Google Cloud Console para cambios de configuración
- Base de datos crecerá rápidamente sin política de limpieza
