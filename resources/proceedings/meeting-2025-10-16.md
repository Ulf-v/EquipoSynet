### Tecnologías debatidas y decididas

**IDEs y Editores:**
- **Visual Studio Code** vs IntelliJ IDEA vs NetBeans
  - VS Code: Ligero, plugins abundantes, gratuito → ELEGIDO
  - IntelliJ: Más pesado, mejor para Java puro pero caro
  - NetBeans: Anticuado, menos usado actualmente
  - **Decisión:** Visual Studio Code para todo (frontend + backend)
  - Plugins que se instalarán: Live Server, Thunder Client/Postman extension, Java Extension Pack, Spring Boot Dashboard

- **Herramientas adicionales:**
  - **MongoDB Compass:** Interfaz gráfica para gestionar MongoDB Server
  - **Postman:** Testing de API REST (confirmado)
  - **.gitBash o PowerShell:** Para terminal Git en Windows (ambos con VS Code)

**Stack definitivo confirmado:**
- **Frontend:**
  - HTML5, CSS3, JavaScript ES6+
  - ~~React~~ Rechazado por tiempo de aprendizaje
  - ~~Vue~~ No considerado
  - **Decisión:** JavaScript vanilla con HTML/CSS puro inicialmente
  
- **Backend:**
  - Java 11 o superior
  - Spring Boot 2.7+ (LTS considerado)
  - Spring Security para autenticación
  - Spring Data MongoDB para integración con BD
  
- **Base de datos:**
  - MongoDB Server (instalada en servidor remoto ya)
  - MongoDB Compass para gestión remota
  - Colecciones iniciales: casos, usuarios, archivos
  
- **Infraestructura:**
  - Docker para contenedorización
  - Docker Compose para orquestación local
  - Nginx como reverse proxy en producción (futuro)
  
- **Autenticación:**
  - Google OAuth 2.0 (confirmado)
  - JWT para tokens de sesión
  - Spring Security 5+

### Puntos tratados  

- **Comparación exhaustiva de IDEs:**
  - VS Code: 6 votos a favor (principalmente por velocidad y plugins)
  - IntelliJ: 2 votos (Jorge y Iván valoran el poder, pero reconocen costo)
  - Result: Consenso en VS Code
  
- **Configuración de MongoDB Server:**
  - Ya instalado en servidor remoto con acceso de equipo
  - Se ha creado usuario de lectura/escritura para dev
  - MongoDB Compass se instalará en cada máquina local
  - Referencias de conexión: mongodb://[servidor]:[puerto]/equiposynet_db
  
- **Validación de compatibilidades:**
  - Java 11 compatible con Spring Boot 2.7+ [Compatibile]
  - MongoDB Server v5.0+ compatible con Spring Data MongoDB [Compatible]
  - Docker Desktop compatible con Windows 10+ [Compatible]
  - Google OAuth 2.0 compatible con Spring Security [Compatible]
  
- **Discusiones técnicas triviales:**
  - **Versionado de dependencias:** Maven vs Gradle
    - Maven: XML, menos flexible pero estándar
    - Gradle: Groovy, más flexible
    - **Decisión:** Maven (pom.xml) por ser más estándar en empresas
    
  - **Testing framework:**
    - JUnit vs TestNG - Juan sugiere JUnit 5
    - Mockito para mocks - confirmado
    - **Decisión:** JUnit 5 + Mockito (aplazado para después de MVP)
    
  - **Formateo de código:**
    - Prettier vs Black (Python)
    - SonarLint en VS Code para java
    - **Decisión:** todos instalen SonarLint extension en VS Code
    
  - **Convenciones de nombres:**
    - CamelCase para variables/métodos
    - PascalCase para clases
    - UPPER_CASE para constantes
    - snake_case para archivos/carpetas
    - **Decisión:** seguir estándares Java/JavaScript comunes

### Decisiones tomadas  
- **IDE:** Visual Studio Code como editor único para todo el equipo
- **Backend:** Spring Boot 2.7+ con Java 11
- **Frontend:** HTML/CSS/JavaScript vanilla (sin frameworks)
- **Base de datos:** MongoDB Server + MongoDB Compass
- **Build tool:** Maven con pom.xml
- **Autenticación:** Google OAuth 2.0 + JWT
- **Versionado:** Java con Maven, JavaScript sin build tools inicialmente
- **Documentación:** README.md en raíz del proyecto

### Riesgos identificados  
- Falta de experiencia en Docker → requiere documentación y tutoriales
- Spring Boot puede tener curva de aprendizaje para principiantes
- MongoDB es NoSQL (diferente a SQL tradicional) → requiere capacitación
- JavaScript vanilla puede ser difícil de mantener sin frameworks estructurados
