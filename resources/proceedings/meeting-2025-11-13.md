### Tecnologías tratadas  
- **Backend:** Spring Boot (Java 11+), estructura de controladores y servicios  
- **Frontend:** HTML/CSS/JavaScript vanilla (sin React finalmente)  
- **API Communication:** RESTful endpoints, content-type JSON  
- **CORS:** Configuración inicial en SecurityConfig.java

### Lo realizado
- Se crea **estructura base del backend** con carpeta `com.synet.demoBack` conteniendo:
  - `DemoBackApplication.java` → clase principal de Spring Boot con @SpringBootApplication
  - `SecurityConfig.java` → configuración inicial de seguridad y CORS habilitado para localhost:3000
  - `LoginController.java` → controlador para endpoints de autenticación
  - Estructura modular para futuros controladores, servicios y repositorios
  
- Se genera **frontend básico** en html/css/js con:
  - `index.nginx-debian.html` → página principal
  - `pages/` → directorio de vistas adicionales (chatbot.html, select_page.html, validacion_Medicos.html)
  - `js/` → scripts de lógica cliente (burger.js, chatbot.js, validation.js)
  - `style/` → hojas de estilos (chatbot.css, style.css)
  
- Se prueba **conexión básica "hola mundo"** entre endpoints:
  - Endpoint GET `/api/test` simple en backend
  - Fetch request GET desde frontend
  - Prueba exitosa tras resolver problemas CORS

### Discusiones técnicas triviales
- **React vs JavaScript vanilla:** Debate largo sobre frameworks. Jorge argumentaba por React por componentes reutilizables, Arón por JavaScript vanilla por simplicidad. Se decide usar JavaScript vanilla como base inicial para acelerar desarrollo.
- **Versiones de Spring Boot:** Discusión sobre versión 2.x vs 3.x. Se elige 2.7+ por estabilidad y documentación.
- **Base de datos:** Confirmación de MongoDB como base de datos (ya instalada en servidor)
- **Librerías frontend:** Consideración de Bootstrap/Tailwind para estilos. Se aplaza decisión a próximas reuniones.

### Problemas encontrados
- CORS policy bloqueando peticiones iniciales desde localhost:3000 → resuelto en SecurityConfig
- Rutas mal configuradas en fetch de frontend → ajustadas con path relativo correcto
- Puerto 8080 por defecto en Spring Boot ya ocupado en máquina de Jorge → reasignado temporalmente

### Decisiones tomadas  
- Crear estructura modular desde el inicio (separación clara entre controladores, servicios, repositorios)
- Frontend basado en HTML/CSS/JavaScript vanilla inicialmente
- Mantener puertos estándar: 8080 (backend en Docker), 3000 (frontend local)
- Documentar en Notion la arquitectura de carpetas
- Crear issues en GitHub para tareas específicas

### Riesgos identificados  
- Falta de experiencia en Spring Boot por parte de Arón y Melisa → mitigado con pair programming
- Posibles conflictos de CORS en despliegue → requerirá revisión cuidadosa en Docker
- Dependencia de documentación online (versiones pueden cambiar)
